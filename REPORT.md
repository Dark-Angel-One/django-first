2. **Исправление кнопки смены темы (Светлая/Темная)**
   - Выявлена проблема: Логика кнопки смены темы находилась в файле `todo_sql/static/js/app.js`, который загружался исключительно на странице авторизованного пользователя (`index.html`).
   - Решение: Скрипт переключения вынесен в конец `todo_sql/templates/todo_sql/base.html`, что обеспечивает его загрузку на всех страницах (включая вход, регистрацию и т.д.).
   - Проверка: Переключатель теперь успешно работает на странице `login.html` и `register.html`.

3. **Кнопка "Вернуться к авторизации" при регистрации**

4. **Адаптация под мобильные устройства**
   - В `base.html` присутствует тег `viewport`, который сообщает браузерам мобильных устройств, как правильно масштабировать страницу.
   - Проверено использование CSS-классов Tailwind: сетка заметок (`.bento-grid`) корректно сворачивается в одну колонку на экранах менее `640px` через адаптивный запрос `@media (max-width: 640px)`. Боковое меню прячется за бэкдропом (реализована функция `closeSidebarMobile()`). Окно создания заметки и модальные формы также адаптивны. Проект корректно работает на мобильных устройствах через браузер.

5. **Поддержка Облачной синхронизации (Cloud Sync)**
   - Запрос пользователя: "если на телефоне и на компьютере открыт один и тот же аккаунт... то без обновления страницы появится и на пк".
   - Реализовано легкое решение в виде периодического опроса (short-polling):
     - В `api_views.py` добавлен метод `check_updates` для `NoteViewSet`, который принимает `last_sync` (время последней синхронизации) и проверяет БД на предмет обновленных или созданных записей после этого времени. Возвращает статус `has_updates`.
     - В `app.js` добавлен интервал раз в 5 секунд, который вызывает этот API. Если приходят обновления, фронтенд неявно для пользователя перезагружает сетку заметок вызовом функции `loadNotes`. Пользователь получает мгновенную синхронизацию на разных устройствах без перезагрузки всей вкладки.

6. **Тестирование системы**
