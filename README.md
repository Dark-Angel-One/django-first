# Клон Google Keep (Todo SQL) - Полное Руководство

Профессиональный, "Pixel-perfect" клон Google Keep, созданный на **Django 5.0+** и **Tailwind CSS**. Этот проект демонстрирует современную "гибридную" веб-архитектуру, органично сочетающую надежность серверного рендеринга Django с высокой динамичностью фронтенда, управляемого API (через Django REST Framework и Vanilla JavaScript).

Проект идеально подходит для тех, кто хочет изучить передовые практики разработки, оптимизации запросов к БД, обеспечения безопасности и создания отзывчивых интерфейсов.

## 🚀 Основной функционал и возможности

### 📝 Заметки и Чеклисты
*   **Два режима**: Создавайте обычные текстовые заметки или переключайтесь в режим чеклиста.
*   **Умные чеклисты**: Динамическое добавление пунктов, интерактивные галочки, автоматическое зачеркивание выполненных задач и оптимистичное сохранение состояния.
*   **Цветовое кодирование**: Визуальная организация заметок с поддержкой палитры из 12 тщательно подобранных цветов, которые идеально смотрятся как в светлой, так и в темной теме.

### 🔐 Аутентификация и Безопасность
*   **Регистрация с подтверждением**: Защита от спама и фейковых аккаунтов. После регистрации на почту отправляется 6-значный код подтверждения. Доступ к приложению предоставляется только после успешного ввода кода.
*   **Смена пароля**: Возможность безопасной смены пароля из меню профиля (с иконками для просмотра скрытого пароля при вводе).
*   **Защита при выходе и удалении**: Модальные окна с подтверждением действий для выхода из аккаунта и безвозвратного удаления профиля (во избежание случайных нажатий).
*   **Кастомизация профиля**: Поддержка загрузки пользовательских аватарок (ImageField) с автоматической обработкой и отображением в шапке.
*   **Обработка ошибок**: Кастомные страницы для ошибок 404 (Не найдено), 403 (Ошибка сессии) и 503 (Сервис недоступен), выполненные в едином стиле приложения (Bento-grid, глассморфизм).

### ⏰ Продвинутые Напоминания
*   **Аудио и Автоматизация**: Напоминания, срабатывающие в течение ближайшего часа, воспроизводят звуковой сигнал и **автоматически отправляют заметку в архив**.
*   **Индикация просроченных**: Если напоминание просрочено более чем на час, оно визуально выделяется (красным индикатором), но заметка **остается на главном экране** для привлечения внимания пользователя.
*   *Техническая реализация*: Клиентский JS опрашивает сервер каждые 30 секунд для проверки статуса напоминаний.

### 🗂 Организация и Порядок
*   **Закрепление (Pin)**: Важные заметки всегда наверху.
*   **Архив и Корзина**: Убирайте неактуальное в архив или удаляйте в корзину (мягкое удаление - soft delete). Корзина не очищается сама, а требует явного действия "Очистить корзину" от пользователя.
*   **Метки (Ярлыки)**: Создавайте метки и привязывайте их к заметкам. Боковая панель мгновенно (реактивно) обновляется при изменении меток без перезагрузки страницы.
*   **Drag-and-Drop**: Перетаскивание заметок мышью для ручной сортировки (при перетягивании в зону закрепленных заметка автоматически получает статус "Закреплено").
*   **Живой поиск**: Мгновенная фильтрация заметок по заголовку и содержимому прямо по мере ввода текста.

### 🎨 UI/UX (Современные Стандарты)
*   **Глассморфизм и Bento-сетка**: Ультрасовременный дизайн. Сетка заметок использует размер страницы равный 12, что идеально делится на 2, 3 и 4 колонки (Bento-grid).
*   **Оптимистичный UI**: Интерфейс реагирует *до* того, как сервер подтвердит изменение. Если вы архивируете заметку, она исчезает мгновенно. При ошибке сети состояние незаметно откатится назад.
*   **Темная/Светлая тема**: Полная поддержка тем с сохранением выбора в `localStorage`. Страницы авторизации также поддерживают глобальную тему.
*   **Адаптивность**: Приложение отлично работает как на огромных мониторах, так и на мобильных устройствах (имеет сворачиваемую боковую панель).

## 🏗 Архитектура и Технологические Решения

Проект использует "Гибридный" подход:
1.  **Django Templates**: Используются для начальной загрузки страницы и SEO. Базовый шаблон (`todo_sql/base.html`) задает скелет, а страницы наследуются от него.
2.  **Django REST Framework (DRF)**: Предоставляет API (`/api/v1/...`) для всех динамических действий (переключение чекбоксов, архивация, закрепление).
3.  **Vanilla JavaScript**: Никаких тяжелых фреймворков вроде React или Vue. Весь интерактив (оптимистичный UI, drag-and-drop, таймеры напоминаний) написан на чистом, высокопроизводительном ES6+ JavaScript.
4.  **Tailwind CSS (CDN)**: Для стилизации используется Tailwind, подключенный через CDN для простоты. Чтобы избежать проблем с кэшированием PWA Service Worker (`sw.js`) настроен особым образом.

### Оптимизация и Безопасность
*   **Борьба с N+1 запросами**: Используется `prefetch_related` для меток и чеклистов, а сериализаторы применяют `bulk_create` и `bulk_update` для пакетной обработки данных.
*   **Производительность БД**: Поля, используемые для фильтрации и сортировки (например, `is_pinned`, `is_trashed`, `order`, `created_at`), имеют индексы (`db_index=True`). Булевы переключатели работают напрямую через `queryset.update()`.
*   **Защита (Security)**:
    *   **IDOR (Insecure Direct Object Reference)**: Строгая проверка того, что пользователь может редактировать только *свои* заметки.
    *   **XSS (Cross-Site Scripting)**: При динамической генерации HTML в JS (например, метки, тосты, пагинация) пользовательский ввод экранируется с помощью `escapeHtml()`, `encodeURIComponent()` или безопасного `textContent`.
    *   Отсутствие хардкода в `SECRET_KEY` и `DEBUG` — они строго читаются из файла `.env`.

## 🗄 Схема базы данных

Приложение `todo_sql` содержит следующие основные модели:

1.  **User** (стандартная модель Django): Владелец всех данных.
2.  **Note (Заметка)**: Главная сущность. Содержит текст, цвет, статус (закреплено/архив/корзина), дату напоминания и порядок сортировки.
3.  **ChecklistItem (Пункт чеклиста)**: Связан с `Note` (ForeignKey). Содержит текст пункта, статус (выполнено/нет) и порядок.
4.  **Label (Метка)**: Связана с `User` и имеет отношение Many-to-Many к `Note`. Обеспечивает тегирование.

## 🛠 Подробное Руководство по Установке и Запуску

Данная инструкция предполагает, что у вас уже установлен Python (версии 3.10 или выше) и Git.

### Шаг 1: Клонирование репозитория
Откройте терминал и выполните команду:
```bash
git clone https://github.com/yourusername/django-keep-clone.git
cd django-keep-clone
```

### Шаг 2: Создание и активация виртуального окружения
Всегда используйте виртуальное окружение, чтобы зависимости проекта не пересекались с вашей системой.
```bash
# Создание (в папке проекта появится директория myvenv)
python3 -m venv myvenv

# Активация для Linux/macOS
source myvenv/bin/activate

# Активация для Windows
myvenv\Scripts\activate
```

### Шаг 3: Установка зависимостей
```bash
pip install -r requirements.txt
```
*(Если вы находитесь в изолированной среде без интернета, этот шаг может завершиться ошибкой "Network is unreachable").*

### Шаг 4: Настройка переменных окружения (.env)
Приложение строго требует наличия файла `.env` с настройками безопасности.
1. Скопируйте файл-пример:
   ```bash
   cp .env.example .env
   ```
2. Откройте файл `.env` в любом текстовом редакторе и обязательно задайте случайную строку для `SECRET_KEY`.
   ```env
   SECRET_KEY=your-very-secret-random-key-here
   DEBUG=True
   ALLOWED_HOSTS=127.0.0.1,localhost
   ```
*По умолчанию база данных будет использовать встроенный SQLite. Для PostgreSQL/MySQL раскомментируйте и заполните поля `DB_ENGINE`, `DB_NAME` и т.д. в файле `.env`.*

### Шаг 5: Сборка статических файлов (Важно!)
Проект использует Whitenoise для раздачи статики. Перед запуском (и перед запуском тестов) **обязательно** выполните сборку статики:
```bash
python manage.py collectstatic --noinput
```

### Шаг 6: Применение миграций базы данных
Создайте таблицы в базе данных:
```bash
python manage.py migrate
```

### Шаг 7: Создание суперпользователя (Администратора)
Для доступа к админ-панели Django (и чтобы просто войти в приложение) создайте аккаунт:
```bash
python manage.py createsuperuser
```
Следуйте инструкциям на экране (введите логин, email, пароль).

### Шаг 8: Запуск сервера разработчика
```bash
python manage.py runserver
```
🎉 Готово! Откройте браузер и перейдите по адресу: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## 🧪 Запуск автоматических тестов

Проект имеет серьезное покрытие тестами, включая проверку бизнес-логики, безопасности (IDOR), API и поведения чеклистов.

Перед запуском тестов убедитесь, что вы выполнили шаг со сборкой статики (`collectstatic`).

Для запуска всех тестов выполните:
```bash
python manage.py test
```

Или запустите конкретные наборы тестов:
```bash
# Тесты безопасности (XSS, права доступа)
python manage.py test todo_sql.tests_security

# Тесты комплексных сценариев (пагинация, индексы)
python manage.py test todo_sql.tests_comprehensive

# Тесты чеклистов и API
python manage.py test todo_sql.tests_checklist
```

## 🔌 Справочник API Эндпоинтов (DRF)

Все API маршруты начинаются с `/api/v1/`. Доступ только для авторизованных пользователей.
API использует `PageNumberPagination` со строгим лимитом в 12 элементов на страницу.

| HTTP Метод | Эндпоинт | Назначение |
|------------|----------|------------|
| `GET` | `/notes/` | Получить список активных заметок (с фильтрацией). Не включает архив и корзину. |
| `POST` | `/notes/` | Создать новую заметку (можно передать вложенные пункты чеклиста). |
| `GET` | `/notes/{id}/` | Получить полную информацию о конкретной заметке. |
| `PATCH` | `/notes/{id}/` | Обновить заметку (смена цвета, текста, заголовка). |
| `DELETE` | `/notes/{id}/` | Безвозвратное (hard) удаление заметки. |
| `POST` | `/notes/{id}/archive/` | Переключатель (Toggle): отправить в архив или вернуть на главную. Возвращает `{'is_archived': true/false}` |
| `POST` | `/notes/{id}/trash/` | Переключатель (Toggle): отправить в корзину или восстановить. |
| `POST` | `/notes/{id}/pin/` | Переключатель (Toggle): закрепить или открепить. |
| `POST` | `/notes/reorder/` | Обновить порядок заметок (принимает списки `pinned_ids` и `other_ids`). |
| `POST` | `/notes/empty_trash/` | Очистить корзину (удалить все заметки со статусом `is_trashed=True`). |
| `GET`/`POST`| `/labels/` | Управление метками пользователя. |
| `PATCH` | `/checklist-items/{id}/`| Изменить состояние пункта чеклиста (галочка, текст). |
